# F2-02 Auth API Key — Design Spec

**Date:** 2026-04-29
**Task:** F2-02 (TASKS.md — Fase 2)
**Effort:** M | **Model:** Opus
**Depends on:** F2-01 (FastAPI scaffold)

---

## Goal

Implement API key authentication for all protected endpoints using FastAPI's
`Security()` + `APIKeyHeader` scheme. API keys are stored as SHA-256 hashes
only; the raw key is returned once at generation time and never persisted.

---

## Approach

**FastAPI `Security()` with `APIKeyHeader`** — idiomatic FastAPI, integrates
with OpenAPI docs (lock icon), uses standard `HTTPException` flow, and is
easily overridable in tests via `app.dependency_overrides`.

---

## Components

### New: `api/app/auth.py`

| Symbol | Type | Description |
|--------|------|-------------|
| `hash_api_key(raw: str) -> str` | function | SHA-256 hex digest of raw key |
| `generate_api_key() -> tuple[str, str]` | function | Returns `(raw_key, key_hash)`. Uses `secrets.token_urlsafe(32)` (256-bit entropy). Raw key never stored. |
| `api_key_header` | `APIKeyHeader` | `APIKeyHeader(name="X-API-Key", auto_error=False)` |

### New: `api/app/schemas/auth.py`

```python
class AuthUser(BaseModel):
    user_id: UUID
    device_id: UUID
    device_label: str
```

### Modified: `api/app/dependencies.py`

Replace `get_current_user` stub:

```python
async def get_current_user(
    raw_key: str | None = Security(api_key_header),
    conn: asyncpg.Connection = Depends(get_db),
) -> AuthUser:
    if not raw_key:
        raise HTTPException(status_code=401, detail="authentication_required")
    key_hash = hash_api_key(raw_key)
    row = await conn.fetchrow(
        "SELECT id, user_id, device_label FROM device_keys "
        "WHERE api_key_hash = $1 AND revoked_at IS NULL",
        key_hash,
    )
    if row is None:
        raise HTTPException(status_code=403, detail="invalid_or_revoked_key")
    return AuthUser(user_id=row["user_id"], device_id=row["id"], device_label=row["device_label"])
```

### Modified: `api/app/routers/` (5 stub routers)

Add `dependencies=[Depends(get_current_user)]` to the `APIRouter(...)` call in:

### Modified: `api/tests/conftest.py`

## Error Semantics

| Situation | HTTP | Detail |
|-----------|------|--------|
| `X-API-Key` header absent | 401 | `authentication_required` |
| Hash not in `device_keys` | 403 | `invalid_or_revoked_key` |
| `revoked_at IS NOT NULL` | 403 | `invalid_or_revoked_key` |

Both 403 cases return the same detail — no information leak about key existence.

---

## DB Query

Single round-trip, no JOIN needed:

```sql
SELECT id, user_id, device_label
FROM device_keys
WHERE api_key_hash = $1
  AND revoked_at IS NULL
```

`last_seen_at` / `last_seen_ip` update is deferred to F2-03 (write path only,
not on every read to avoid write amplification).
