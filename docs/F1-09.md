## Contract mDNS

```
Service type:   _memorymesh._tcp.local.
Instance name:  MemoryMesh on <host>.<type>
Port:           80 (Caddy LAN) / 443 (Caddy VPN/public)
TXT record:     version=<x.y>  service=memorymesh
```

Il plugin Claude Code + adapter Codex cercano **tutte** le istanze di
`_memorymesh._tcp.local`, filtrano per `version` nel TXT record per
matching compatibile, e usano `port` + `server` (AAAA record) per
costruire la URL base `http://<server>:<port>/`.

## Decisioni di design

### 1. `InterfaceChoice.Default` invece di `None`

`zeroconf==0.135` **richiede** `InterfaceChoice.Default` o una lista di
IP quando non si filtra per interfaccia. `None` fa raise `TypeError:
choice must be a list or InterfaceChoice, got None`.

### 2. `IPVersion.V4Only` di default

IPv6 aggiunge complessitĂ  nel matching plugin (dual-stack LAN). Il
fallback IPv4 Ă¨ universale e sufficiente per LAN home/office.

### 3. Auto-detect IPv4 senza internet egress

`_resolve_local_ipv4()` apre un socket UDP verso `8.8.8.8:53` e legge
`getsockname()[0]` â€” trick standard che funziona anche **offline**
(il kernel risolve l'IP dell'interfaccia outbound senza inviare il
pacchetto). Fallback `127.0.0.1` se OSError.

### 4. TXT record: `bytes` mandatory

zeroconf 0.135 vuole le properties come `dict[bytes, bytes]` (non
`dict[str, str]`). Il plugin deve decodificare UTF-8 lato client.

### 5. Instance name "MemoryMesh on \<host>"

Leggibile da admin UI e `avahi-browse`. Hostname splittato su `.` e
preso solo il primo segmento â€” `MemoryMesh on mm` invece di
`MemoryMesh on mm.local` (che sarebbe redundant).

### 6. Server field con suffix `.local.`

Lo `server` di `ServiceInfo` Ă¨ il nome AAAA/A record mDNS. Deve
terminare con `.local.` (trailing dot = FQDN absolute). Il broadcaster
normalizza: se l'utente passa `"mm.local"` â†’ normalizza a `"mm.local."`;
se passa `"mm"` â†’ `"mm.local."`.

## Gotcha & portabilitĂ 

1. **Docker Desktop Mac/Win + `network_mode: host`**: il container NON
   vede la LAN vera. mDNS multicast non esce dal VM. Soluzione: avahi
   con `AVAHI_REFLECTOR=yes` â€” oppure eseguire zeroconf sul **host** e
   container su rete bridge.
2. **Windows Python path escaping**: quando il test script passa un
   path Windows a heredoc Python (`ET.parse("$PATH_PY")`), le
   backslash diventano escape (`\a`, `\t`, `\n`). Soluzione: usare
   `cygpath -m` per ottenere forward-slash (`G:/workspace/...`).
3. **zeroconf 0.135 API**: `interfaces=None` non Ă¨ valido â€” usare
   `InterfaceChoice.Default`.
4. **Multicast su bridge Docker interno**: `mm_internal` (internal:true)
   non instrada multicast verso LAN. Serve bridge non-internal o
   network_mode host per propagazione reale.

## Integrazione con F2 (preview)
